While ranger has amazing features and is the patron saint of TUI file managers, alternatives have been emerging with a greater emphasis on speed and minimalism. This is one of those attempts as sort of a pet project and something to do for fun. The project itself can be found here.

Approach

My main goal was to imitate and improve upon another ranger alternative called cfiles. Initially, I wanted to take the existing source code and improve the design of it as well as simplify it a bit more. However, I felt that that would take more work than simply starting from scratch.

I wanted to have something that could display directory content, support file previews as well as navigate to and open files. I also wanted to have something highly configurable and easily modifiable either in source code or using scripts called from the application. Another good thing to have is image previews in the terminal as well as the ability to extend the application easily with user modules.

Current Status

At the moment, aside from some bugs, the application performs what I wanted from a minimal file manager and is in a fairly good place. I have more ambitious goals laid out on the GitHub page that I may address in the future.
