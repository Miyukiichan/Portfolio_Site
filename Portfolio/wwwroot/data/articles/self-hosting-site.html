As briefly outlined in the original article involving this site, the free Azure hosting solutions are not very good for even personal sites. The main issue is to do with the initial loading time of the site, regardless of complexity. As a result, I decided to host the site myself which was a somewhat involved but fairly straightforward process. Microsoft has detailed documentation on this which makes the process easier.

Hosting

I decided to use Vultur as a hosting provider. The reason for this was that I had seen other projects running it before and knew that it would be sufficient for my purposes. Also, the pricing is very cheap for how capable it is. The other advantage to using a server like this is that I can just manage it like I would any normal Linux system as opposed to using a labyrinthine web client. In addition to this, a server like this can easily be configured to host multiple domains at no extra cost on the server end which is great.

Server

The server is being hosted using Nginx which is easy to configure and maintain. I also wanted an excuse to learn Nginx as I have had some experience with Apache in the past. For https support, I chose to use Certbot to generate the correct Nginx configuration as well as register the domains correctly.

Results

Using a dedicated custom server has certainly paid off in comparison to the free Azure hosting solutions. This in terms of cost as well as stability, configurability and performance.
